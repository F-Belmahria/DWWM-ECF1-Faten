1. Présentation du projet

Le Phosphore est une salle de spectacle proposant du théâtre, des concerts, du stand-up et divers événements culturels.
Le projet consiste à concevoir un site web moderne, accessible et responsive permettant de présenter la salle, sa programmation et les informations pratiques.
2. Public cible

public de 25 à 55 ans ;
utilisateurs sur mobile, tablette et ordinateur ;
tests prévus sur iPhone, Samsung et Pixel.
3. Attentes du client

design chaleureux et sophistiqué ;
ambiance liée au spectacle vivant ;
navigation simple ;
site responsive ;
accessibilité RGAA ;
animation d’ouverture des rideaux.
4. Besoins des utilisateurs

Les visiteurs doivent pouvoir :

découvrir la salle ;
consulter les spectacles ;
filtrer les spectacles ;
voir les dates et les places disponibles ;
consulter les tarifs ;
trouver les informations d’accès ;
consulter les services et l’accessibilité PMR.
5. Contraintes graphiques

deux polices maximum ;
police serif pour les titres ;
police sans-serif pour les textes ;
quatre couleurs seulement ;
bordeaux, or, anthracite et crème ;
aucun noir pur ni blanc pur.
6. Contraintes techniques

trois pages minimum ;
HTML ;
CSS et Sass ;
méthode BEM ;
JavaScript ;
fichier JSON ;
responsive mobile first ;
Git et GitHub ;
publication HTTPS ;
validation W3C et ESLint.
7. Éléments à réaliser

Quantifiez le projet :

3 pages minimum ;
8 spectacles ;
8 cartes de spectacle ;
1 fichier JSON ;
1 animation de rideaux ;
3 filtres par type ;
plusieurs filtres par date ;
1 filtre de disponibilité ;
1 barre de progression par spectacle ;
4 couleurs ;
2 polices maximum ;
120 places par spectacle.
8. Programme de la semaine

Répartissez les 35 heures :

Jour 1 : analyse et organisation — 7 h
Jour 2 : maquettes Figma — 7 h
Jour 3 : HTML et Sass — 7 h
Jour 4 : JavaScript, JSON et animation — 7 h
Jour 5 : tests, corrections et publication — 7 h
